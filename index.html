중간 산출물 처리 방식
자동 통과
중간 결과물을 검토 없이 다음 단계로 바로 넘겨요.
API 호출 최소
오류가 나도 나중에 최종 결과물에서 잡아낼 수 있을 때 써요. 예) 파일 포맷 변환, 단순 데이터 합치기.
자동 검증 후 통과
LLM이 중간 결과물을 확인하고 이상 없으면 자동으로 넘어가요.
API 호출 +1
중간 결과물 품질이 다음 단계에 영향을 줄 때 써요. 예) 요약 품질 확인, 데이터 누락 체크.
사람 승인 후 통과
중간 결과물을 사람이 확인하고 승인해야 다음 단계로 넘어가요.
대기 시간 발생
판단 분기점이거나 오류가 뒤에서 수정 불가능할 때. 예) 보고서 초안 검토, 분류 결과 확인.
단계마다 달라요
LLM이 각 단계 성격을 보고 자동/검증/승인을 직접 결정해요.
가장 정밀한 설계
단순변환=자동통과, 품질영향=자동검증, 판단분기점=사람승인으로 LLM이 단계별로 최적 방식을 판단해요.
